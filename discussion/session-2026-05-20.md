### 2. Analyse de l'ancien modèle PyWebView

Analyse complète des templates HTML de l'ancienne version (`F:\KS_Production_PyWebView\KS_Production\templates\`) pour identifier :
- Les fiches/aperçus/reçus à reconduire fidèlement
- Les formulaires de saisie à repenser visuellement

**Fiches identifiées :**
- `apercu_facture.html` — Document A4 avec en-tête, tableau, totaux, cachet/signature
- `apercu_proforma.html` — Idem + filigrane PROFORMA + badge non contractuel
- `apercu_recu.html` — Reçu compact, barre de progression paiement
- `apercu_prestation_pdf.html` — Fiche équipe + matériel + facture liée
- `apercu_relance_pdf.html` — Lettre officielle, montant dû en rouge

**Formulaires identifiés :**
- `nouvelle_facture.html` — Sidebar + main, lignes services éditables, récap dynamique
- `nouvelle_proforma.html` — Variante violette
- `enregistrer_paiement.html` — Boutons montants rapides, aperçu avant validation
- `nouveau_client.html` — Avatar temps réel, infos fiscales
- `nouvelle_operation.html` — Sélecteur Recette/Dépense

### 3. Décision sur le format des formulaires

**Comparaison visuelle présentée :**
- Option A : Modal popup (compact)
- Option B : Page dédiée (espace illimité)

→ Maxime a d'abord choisi Option B (page dédiée), puis a précisé que les formulaires doivent être dans des **fenêtres flottantes** qui s'ouvrent sur la fenêtre principale.

**Résultat :** Modal large 2 colonnes validée (voir `public/demo-modal-facture.html`)

### 4. Maquette formulaire Nouvelle Facture validée

Structure de la modal validée :
- **Header** : icône + titre + numéro auto + Aperçu PDF + fermer
- **Colonne gauche (blanche)** : champs généraux (numéro readonly, date, mode paiement, client avec autocomplétion, boutons section), tableau de lignes de services éditables, suggestions rapides cliquables
- **Colonne droite (fond sombre #1a1a2e)** : récapitulatif financier temps réel, zone acompte + reste dû, notes internes, bouton Enregistrer + Annuler

Fichier de référence : `public/demo-modal-facture.html`

---

## Décisions de design importantes

| Règle | Détail |
|-------|--------|
| **Formulaires** | Fenêtre flottante (modal) 2 colonnes, jamais de page dédiée |
| **Couleurs** | Fond blanc dominant, couleurs uniquement en accents/badges |
| **Navigation** | Sidebar accordéon au clic, pas de flyout au survol |
| **Admin** | Visible seulement si `role === 'Administrateur'` |
| **Polices** | Geist Sans + Geist Mono (alternatives rejetées) |
| **Fiches/PDF** | Reconduire fidèlement l'ancien modèle PyWebView |
| **Workflow** | Visual first → validation Maxime → code ensuite |
